# Конспект. HW-0: Настройка окружения и первый опыт с агентом

## 1. Настройка окружения

### Установка инструментов
- Всё ставится через `make` из `.ai-setup/` — mise, node, ruby, gh, direnv, агенты, скиллы
- `mise` — менеджер версий инструментов, ставит всё изолированно в `~/.local/share/mise/`
- Ruby через mise не конфликтует с rbenv — переключается по директориям (где есть `mise.toml`)

### Проблема с direnv
- `eval "$(direnv hook zsh)"` не работал — на момент вызова mise ещё не добавил direnv в PATH (ленивая активация)
- Решение: вызов по абсолютному пути `eval "$("$HOME/.local/share/mise/installs/direnv/latest/direnv" hook zsh)"`
- Урок: порядок инициализации в `.zshrc` важен

### Проверка
- `bash scripts/test-setup.sh` — проверяет всё: инструменты, авторизацию, shell integration
- Итог: 0 failures

## 2. Чистый контекст — ключевая идея курса

### Что говорит преподаватель
- Чем меньше скиллов/агентов/MCP — тем лучше
- MCP засоряют контекст и заменяются CLI-утилитами
- Даже неактивные скиллы смещают "мышление" модели
- Правило: всё отключено на уровне системы, включать только под проект/задачу

### Что сделали
- Глобальный `~/.claude/CLAUDE.md` — отсутствует (чисто)
- Проектный MCP: только context7
- В CLI-режиме (`claude` в терминале) загружаются только проектные настройки

### Мониторинг контекста
- `ccstatusline` — постоянный индикатор в статус-строке (токены, модель, ветка)
- `/statusline` — встроенная команда Claude Code, но запускает агента и тратит контекст (~6.6k токенов на ничего)
- Накладные расходы ccstatusline — ноль токенов (работает вне агента)

## 3. PROJECT.md — описание проекта

### Требования преподавателя
- Что за проект, для кого, зачем
- 2-3 абзаца прозой
- БЕЗ бэклога задач, implementation details, пронумерованных списков
- БЕЗ разговорного стиля
- Архитектурные решения — в CLAUDE.md, не в PROJECT.md

### Итог
- Mock-сервер для эмуляции внешних API
- Для QA-команды
- Решает проблему: нет демо-сред, креденшелы протухают, невозможно покрыть все сценарии

## 4. CLAUDE.md — контекст для агента

### Структура
- Stack: Ruby 3.4.8, Rails, SQLite, Minitest, Rubocop
- Key commands: server, test, migrate
- Conventions: Rails-way + простота, валидации вне моделей, сервисы по мере усложнения
- Constraints: не добавлять гемы, не менять БД, не трогать файлы вне scope

### Обсуждение стека
- Rails-way vs слоёная архитектура — начинаем с простого, усложняем по мере необходимости
- Валидации вне моделей — принципиальное решение
- Service objects — по мере появления бизнес-логики
- SQLite вместо PostgreSQL — для старта достаточно

## 5. Промпт для агента

### Формат
- Goal → Context → Requirements → Acceptance
- Описывать ЧТО нужно пользователю, а не КАК реализовать
- Не дублировать информацию из CLAUDE.md
- Implementation details (структура таблиц, роуты) — пусть агент решает сам

### Первый промпт
- Базовая аутентификация
- CRUD для моков (url, метод, заголовки, тело, статус-код)
- Клиент может отправить запрос и получить ожидаемый ответ

## 6. Первый опыт с агентом

### Наблюдения при запуске
- Агент выбрал Rails 7.2 хотя в CLAUDE.md было Rails 7 — пришлось поправить на Rails 8
- Уровни effort: medium (стандартный) vs max (глубже, дороже)
- Сессии Claude Code сохраняются — можно вернуться через `claude --resume <id>`
- Но лучше новая сессия — чистый контекст
