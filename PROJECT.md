# Mock Server для внешних интеграций

Конфигурируемый mock-сервер для эмуляции внешних API, с которыми интегрируется основной продукт. Предназначен для QA-команды, которая тестирует фичи — как простые, так и пошаговые.

Проблема: не у всех внешних сервисов есть демо-среды, креденшелы протухают после интеграции, а покрыть все сценарии (ошибки, таймауты, edge-кейсы) на реальных API невозможно. Тестировщикам приходится зависеть от доступности внешних сервисов и привлекать разработчиков для настройки окружения.

Mock-сервер позволяет поднимать заглушки внешних API и управлять их поведением: переключать сценарии ответов, эмулировать ошибки и таймауты — без зависимости от реальных внешних сервисов.
